# Linear Diffs

A code-review surface built into [Linear](index.md), launched **May 28, 2026**. It
lets teams "review diffs from any issue with a PR, iterate on further changes with
agents, and ship code from Linear."

## Overview

- **Vendor:** Linear.
- **Product:** Linear Diffs — launched May 28, 2026.
- **Category:** code review / PR walkthrough inside a work-tracking tool.

## Features

- **Structural diffing** — view unified or split diffs with line wrapping,
  structural highlighting, custom code themes, and a file list.
- **Guided reviews (beta)** — walks reviewers through large PRs by presenting the
  **core of the change first** with per-section explanations, separating out glue
  code and secondary changes.
- **Reviews inbox** — a Reviews tab for review requests, your approved PRs, and
  comment mentions, sorted by what's closest to shipping.
- **Agent iteration** — keep iterating from the diff surface with a **background
  coding agent**; agent changes update the diff in real time.
- **GitHub sync** — reviews sync back to GitHub so review state stays consistent.

## Pricing

**Available on all plans.** Guided reviews are **free during beta** and
afterwards available to **Business and Enterprise** customers. Setup requires a
workspace admin to upgrade Linear's GitHub integration before users can enable
diff reviews.

## Competitive surface

- **Guided reviews (semantic-order PR walkthroughs)** — organizing a PR into
  explained, ordered sections to walk a human through changes is the point of
  closest overlap with Principal's trail-based PR review.
- **Review + agent iteration in one surface** — reviewing and re-driving a coding
  agent from the same place touches Principal's review/collaboration domain.

How these compare to Principal is a trail concern, not recorded here.
