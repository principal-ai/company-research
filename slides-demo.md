# Principal AI

### Code Trails — answers that carry their own evidence

A research substrate that makes every fact **pointable**, so the
answer *is the path*.

---

## The Problem

Engineering knowledge rots the moment it's written down.

- Docs drift from the code they describe
- "Why is it worth this?" has no auditable answer
- Onboarding repeats the same archaeology every time

> The answer exists — it's just not *grounded* in anything that stays true.

---

## The Mechanism

Two halves of one idea:

1. **OKF** — one concept per doc, stable anchors, a source + as-of date on
   anything that can rot. Facts become *pointable*.
2. **Trails** — an ordered walk of markers, each pinned to one canonical fact.
   The walk **is** the answer.

Facts stay neutral; the argument lives in the trail.

---

## Why It's Different

```text
fact base  ──►  honest internal trail
           ──►  bullish investor trail
```

The same neutral facts carry *both* trails — the docs never take a side, so
the evidence stays honest as it updates.

---

## Traction Surfaces

- **product/** — what Code Trails is and how it's designed
- **competitors/** — one neutral doc per rival
- **market/** — dated, sourced external signal
- **investors/** — thesis + fit, one fund per doc

Each value argument is a trail *across* these folders, not a folder itself.

---

## The Ask

We're raising to turn pointable facts into the default way teams answer
**"what is this worth?"** — at every altitude.

- Product value
- Competitive value
- Market value
- The investment case
